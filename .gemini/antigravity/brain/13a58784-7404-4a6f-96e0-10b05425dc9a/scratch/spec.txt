Complete System WorkflowThe following describes the exact end-to-end flow a user experiences and that the system must support.4.1 Step-by-Step User FlowUser lands on EYES web app. Sees a clean dark interface with two buttons: Connect Reddit and Connect Google.User clicks Connect Reddit. Browser redirects to Reddit OAuth page. User approves permissions. Reddit returns an access token to the EYES callback URL.App stores the Reddit access token securely. Immediately begins data ingestion in the background.App calls Reddit API — GET /user/{username}/comments and GET /user/{username}/submitted. Paginates through full history using the 'after' parameter. Stores every item in Supabase with metadata.User optionally clicks Connect Google. Google OAuth flow runs. App receives access to Gmail API and Google Calendar API.App pulls Gmail messages using the Gmail API. Pulls Calendar events using Calendar API. Stores all items in Supabase.Embedding pipeline runs. Every stored item gets converted into a vector using an embedding model. Vectors stored in Supabase pgvector table alongside raw text.User sees live indexing counter. When complete, counter shows: "17,261 memories indexed — ready."Chat interface appears. User types: "What did I think about AI in 2022?"The question gets embedded into a vector. Vector similarity search runs against all stored memory vectors. Top 10 most semantically relevant items returned.Those 10 items sent to Claude API as context alongside the user's question.Claude generates a response drawing only from the provided context. Response includes source attribution.User sees the answer in the chat interface with citation chips below.4.2 System Architecture DiagramThe system has five independent components that connect sequentially:USER BROWSER

CREATE EXTENSION IF NOT EXISTS vector;6.2 Users TableCREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  reddit_username TEXT,
  google_email    TEXT,
  reddit_token    TEXT,    -- encrypted access token
  google_token    TEXT,    -- encrypted access token
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  last_sync_at    TIMESTAMPTZ
);6.3 Memories TableCREATE TABLE memories (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id) ON DELETE CASCADE,
  platform    TEXT NOT NULL,     -- 'reddit' | 'gmail' | 'gcalendar'
  type        TEXT,              -- 'comment' | 'post' | 'email' | 'event'
  content     TEXT NOT NULL,     -- full raw text
  url         TEXT,              -- link back to original
  metadata    JSONB,             -- subreddit, subject, attendees etc
  posted_at   TIMESTAMPTZ,       -- when the item was originally created
  embedding   vector(1536),      -- OpenAI text-embedding-3-small dimension
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Index for fast vector search
CREATE INDEX ON memories USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);6.4 Vector Search FunctionCreate this SQL function in Supabase for similarity search:CREATE OR REPLACE FUNCTION match_memories(
  query_embedding vector(1536),
  match_user_id   UUID,
  match_count     INT DEFAULT 10
)
RETURNS TABLE (id UUID, content TEXT, platform TEXT,
               url TEXT, posted_at TIMESTAMPTZ, similarity FLOAT)
LANGUAGE sql STABLE AS $$
  SELECT id, content, platform, url, posted_at,
         1 - (embedding <=> query_embedding) AS similarity
  FROM memories
  WHERE user_id = match_user_id
  ORDER BY embedding <=> query_embedding
  LIMIT match_count;
$$;7. API Routes to BuildAll routes are Next.js API routes under /app/api/.
